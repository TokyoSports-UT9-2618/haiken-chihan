# 廃県置藩シミュレーター — 設計書 v2.0
> 作成: 2026-03-21 / 対象: 全国版フル展開

---

## 1. コンセプト

> 現代の市区町村を「藩（スマートシュリンクシティ）」にグループ化する行政再編シミュレーター。
> プレイヤーは地理・人口・財政を見ながら、理想の自治体連合を設計する。

**目標人口**: 1藩あたり 30〜50万人（中心値40万）
**ゲームのゴール**: 選択エリア内の全市区町村をいずれかの藩に割り当て、スコアを競う

## 3. ゲームロジック

### 藩の作成フロー

```
1. 「新しい藩を作る」ボタン
   → state.hanCounter++ → 色を自動割当（HAN_COLORS 10色ローテ）
   → state.currentHanId にセット

2. 地図上の市区町村をクリック
   → canAddToHan() で隣接チェック（BFS連結性保証）
   → 割当済みならトグルで外す

3. 「この藩を確定」
   → 名前入力ダイアログ
   → 人口 < 30万 → 特区(tokku)ダイアログ（減点半減）
   → state.hans[].confirmed = true

4. 全市区町村割当完了 → スコア計算 → クリア画面
```

### 隣接制約（canAddToHan）

```javascript
// 1. 藩メンバーが0人 → 無条件OK
// 2. 追加しようとする市区町村が、藩メンバーの少なくとも1人と隣接している → OK
// 3. それ以外 → NG（飛び地禁止）
// ※ 連結性は追加時のみチェック（削除時は「削除後も連結か」をBFSで確認）
```

### スコア計算

```
ベース: 藩ごとに1000点
人口加減点:
  30〜50万: ±0（満点）
  25〜30万 / 50〜55万: -100
  20〜25万 / 55〜60万: -200
  〜20万 / 60万〜: -400
  特区あり: 減点を半分に
藩数ボーナス:
  市区町村数 / 10 以上の藩数 → +500/藩
```

### 行政コスト試算

```
BEFORE（現状）:
  知事・副知事報酬 + 県議会費 + 県施設費
  + 全市区町村の（首長報酬 + 議員報酬 + 施設費）

AFTER（廃県置藩後）:
  確定した藩ごとに:
    藩主報酬（首長規模で算出）
    + 議員報酬（削減率スライダーで調整）
    + 施設費（統廃合率スライダーで調整）
```

#### 「範囲を広げる」ボタンの動作

```
1. 現在の状態を自動保存（localStorage autosave）
2. landing.html?expand=1&prefs=04,07 にリダイレクト

  現在の選択県 → 確定（濃色、解除不可）
  隣接する未選択県 → 薄色（追加可能）
  それ以外 → グレー（選択不可）

  表示: 「現在のエリアに追加する県を選んでください」
  ボタン: [追加してゲーム再開]（選択なしなら無効）

3. 追加県選択 → [追加してゲーム再開]
   → index.html?prefs=04,07,06&restore=autosave

index.html:
  autosaveを復元（hans, assignments）
  新しい県のデータを追加ロード
  新しい市区町村は「未割当」で地図に追加表示
  既存の藩はそのまま保持
```

## 10. 主な既知の課題

| 課題 | 重要度 | 対処方針 |
|------|--------|----------|
| 税収データが福島のみ | 中 | ESTAT_APIキー取得後に generate-all で一括取得 |
| 東京23区・政令市の区の表示 | 高 | e-Stat GISに区レベルデータあり（generate済みで対応可） |
| 離島（沖縄を除く）の隣接なし問題 | 中 | 隣接数0の市区町村を「どの藩にも入れられる」扱いに |
| 選択エリアが広いとき地図の初期ズームが外れる | 低 | 全市区町村のBBoxからfitBounds()で自動調整 |
| R2へのGeoJSONアップロード自動化 | 低 | wranglerスクリプト化（generate-all.jsのpost処理に追加）|
