## 1. 追加するデータ構造

### 1-1. 県レベルのコストデータ

```javascript
const PREFECTURE_COST = {
  name: "福島県",
  // 知事・副知事（特別職給与。概算：月額約120万円×12＋期末手当で年1,800万円）
  governor: {
    salary_annual: 18000000, // 円/年（概算）
    count: 1
  },
  // 県議会議員（定数58人、月額約99万円、期末手当含め年1,700万円/人）
  assembly: {
    salary_annual: 17000000, // 円/年/人（概算）
    count: 58
  },
  // 県庁舎・出先機関の維持費（概算：年約30億円）
  // ※福島県公共施設等総合管理計画より推計
  facility_annual: 3000000000, // 円/年
  
  // 計算用サマリ
  get total_annual() {
    return (
      this.governor.salary_annual * this.governor.count +
      this.assembly.salary_annual * this.assembly.count +
      this.facility_annual
    );
  }
  // ≒ 1,800万 + 9億8,600万 + 30億 ≒ 約41億円/年
};
```

### 1-2. 市町村レベルのコストデータ（59市町村）

```javascript
// MUNICIPALITIES配列に cost フィールドを追加
// 各市町村の首長報酬・議員報酬・役場維持費を人口に応じて概算
// 算出ロジック：
//   首長報酬 = 人口規模に応じた5段階（下表参照）
//   議員数   = 人口÷8,000人（総務省の人口比例モデル）を四捨五入、最低4人
//   議員報酬 = 人口規模に応じた4段階
//   役場維持費 = 人口×1万円/人/年（小規模自治体は最低3,000万円）

const SALARY_TIERS = {
  // 首長月額報酬（人口規模別）
  mayor: [
    { minPop: 200000, monthly: 1000000 }, // 20万人以上（郡山・いわき・福島クラス）
    { minPop:  50000, monthly:  800000 }, // 5〜20万人（中核市相当）
    { minPop:  20000, monthly:  650000 }, // 2〜5万人
    { minPop:   5000, monthly:  500000 }, // 5千〜2万人
    { minPop:       0, monthly: 380000 }, // 5千人未満（村・小町）
  ],
  // 議員月額報酬（人口規模別）
  member: [
    { minPop: 200000, monthly: 570000 }, // 20万人以上
    { minPop:  50000, monthly: 380000 }, // 5〜20万人
    { minPop:  20000, monthly: 270000 }, // 2〜5万人
    { minPop:       0, monthly: 200000 }, // 2万人未満
  ]
};

// 算出関数
function calcMunicipalityCost(pop) {
  const mayorTier = SALARY_TIERS.mayor.find(t => pop >= t.minPop);
  const mayorAnnual = mayorTier.monthly * 12 * 1.4; // 期末手当含む
  
  const memberCount = Math.max(4, Math.round(pop / 8000));
  const memberTier = SALARY_TIERS.member.find(t => pop >= t.minPop);
  const memberAnnual = memberTier.monthly * 12 * 1.4 * memberCount;
  
  const facilityAnnual = Math.max(30000000, pop * 10000); // 最低3,000万
  
  return {
    mayorAnnual,
    memberCount,
    memberAnnual,
    facilityAnnual,
    total: mayorAnnual + memberAnnual + facilityAnnual
  };
}
```

## 4. 実装ステップ（Claude Codeへの指示順）

### Step 1：データ層の追加

既存の `MUNICIPALITIES` 配列に `cost` フィールドを追加。
`calcMunicipalityCost(pop)` 関数で自動計算してハードコード。

```javascript
// 追加するフィールドイメージ
{ code: "07201", name: "福島市", pop: 280112, ..., cost: {
  mayorAnnual: 16800000,
  memberCount: 35,
  memberAnnual: 188160000,
  facilityAnnual: 2801120,
  total: 207761120
}}
```

### Step 2：`state` にコスト集計フィールドを追加

```javascript
const state = {
  // 既存フィールドはそのまま
  ...
  // 追加
  currentCosts: {
    before: {}, // 廃県置藩前コスト（固定値）
    after: {},  // 現在の藩編成に基づくコスト（リアルタイム計算）
  },
  sliders: {
    memberReductionRate: 0.5,  // 議員削減率
    facilityClosureRate: 0.4,  // 施設廃止率
  }
};
```

### Step 3：クリア画面にコスト比較UIを追加

既存のクリア画面 (`showClearScreen()` 相当) に、コスト比較ブロックを追加。
スライダーは `<input type="range">` で実装し、`oninput` でリアルタイム更新。

### Step 4：サイドパネルにコストプレビューを追加

藩の人口ゲージの下に、現在編集中の藩のコスト概算を表示。
